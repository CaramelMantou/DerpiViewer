## Project Overview

Derpiviewer is a Flutter-based Android application for browsing image boards (boorus) powered by the Philomena software. Supports 7 booru hosts, search with configurable filters, infinite-scroll trending, full-screen gallery with slideshow, local SQLite favorites, download/share, dark mode, and EN/ZH localization.

## Quick Reference

- **Tech Stack:** Flutter 3.32.7 / Dart 3.8.1 / Provider / Dio / sqflite
- **Entry Point:** `lib/main.dart:13` — `main()`
- **Architecture Pattern:** Provider + ChangeNotifier (MVVM-like)
- **Database:** SQLite via sqflite (`favourites` table)
- **Deployment:** Android APK (`flutter build apk --split-per-abi`)

### Known Architecture Issues (See architecture.md for full details):

1. **God file** `enums.dart` — mixes config, UI colors, and l10n logic
2. **Singletons** — `BasePhilomenaClient`, `DioClient`, and static `DbHelper` prevent testing
3. **No repository layer** — Models directly call API and DB
4. **No tests** — Zero unit, widget, or integration tests
5. **ImageResponse dual role** — Single class serves as API DTO, DB entity, and UI model

### Refactoring Path (aligns with user goals):

1. Split `enums.dart` → `config/`, `theme/`, separate enums
2. Introduce repository interfaces for API and DB access
3. Add dependency injection (e.g., `get_it`) to replace singletons
4. Separate `ImageResponse` into API DTO and domain entity
5. Extract `SearchInterface` to its own file
6. Split `home_page.dart` into separate widget files
7. Add unit tests as refactoring enables testability
