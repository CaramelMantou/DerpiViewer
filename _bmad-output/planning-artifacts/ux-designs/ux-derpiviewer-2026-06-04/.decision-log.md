## Decisions

1. **Theme system:** Dual-theme (light/dark) via Material `ThemeData` with `AppTheme` static class. Switched via `PrefModel.isDarkMode`.
2. **Color palette:** Light uses `Colors.blue` primary; dark uses `Colors.blueGrey` primary. Tag colors are hardcoded ARGB constants in `ConstStrings`.
3. **Typography:** Single font family (Material default — Roboto). No custom fonts. Size hierarchy: 12 (subtitle), 14-16 (body), 18 (headings in detail), 20 (AppBar title).
4. **Layout:** Mobile portrait, single/dual column grid toggle, no tablet/landscape adaptations.
5. **Navigation:** Stack-based (Navigator.push). No named routes, no deep linking.
6. **Gesture primitives:** Tap (primary), swipe (gallery page turns), pinch-to-zoom (gallery). No pull-to-refresh, no long-press.
7. **State feedback:** Loading indicators (CircularProgressIndicator), error icons (Icons.error, Icons.error_outline). Toast messages for actions (fluttertoast). No skeleton screens.
8. **i18n:** EN + ZH via `flutter_localizations` + `intl`. Hardcoded Chinese strings in drawer for unimplemented keys.
9. **Accessibility:** Minimal — basic tooltips on some IconButtons. No semantic labels or screen-reader-specific markup.
10. **No brand identity:** No logo, no custom iconography, no motion design beyond defaults.
